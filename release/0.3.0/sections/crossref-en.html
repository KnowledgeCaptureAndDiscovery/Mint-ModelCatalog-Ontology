Model configurationc back to ToC or Class ToC

IRI: https://w3id.org/mint/modelCatalog#ModelConfiguration

A model may be configured in different ways depending on the data that is avaialble. For example, if topological information is available, a groundwater model may use it to produce a better estimation of the water budget of an aquifer. A model configuration represents a unique setting of inputs and outputs of a given model designed for a particular functionality.
has super-classes
function c
is in domain of
has CAG op, has input op, has module op, has output op, has parameter op, has process op, has rule dp, has time interval op
is in range of
has configuration op

Variable presentationc back to ToC or Class ToC

IRI: https://w3id.org/mint/modelCatalog#VariablePresentation

Concept used to represent an instantiation of a variable in an input/output dataset. For example, a model A may use an input file with temperature expressed in Farenheit (variablePresentation1), while a model B may produce an output with temperature in Celsius (variablePresentation2). Both variable presentations refer to the concept of temperature.
has super-classes
Variable c
is in domain of
has dataset op, has long name dp, has short name dp, has standard variable op
is in range of
has presentation op
